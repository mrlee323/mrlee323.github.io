## Container Queries란?

기존 미디어 쿼리(`@media`)는 뷰포트 크기에 반응합니다. Container Queries는 **부모 컨테이너의 크기**에 반응합니다.

이것이 왜 중요한가요? 같은 카드 컴포넌트가 사이드바에서는 좁게, 메인 콘텐츠 영역에서는 넓게 표시될 때, 뷰포트 크기로는 이를 구분할 수 없습니다.

## 기본 문법

```css
/* 컨테이너 정의 */
.card-wrapper {
  container-type: inline-size; /* 가로 크기만 추적 */
  container-name: card;        /* 선택적 이름 */
}

/* 또는 단축형 */
.card-wrapper {
  container: card / inline-size;
}

/* 컨테이너 크기에 따른 스타일 */
@container (min-width: 400px) {
  .card {
    display: grid;
    grid-template-columns: auto 1fr;
  }
}

@container card (min-width: 600px) {
  .card {
    padding: 2rem;
    font-size: 1.125rem;
  }
}
```

## 실전 예제: 적응형 카드

```tsx
// ProductCard.tsx - 컨테이너 크기에 따라 레이아웃 변경
function ProductCard({ product }: { product: Product }) {
  return (
    <div className="product-card-wrapper">
      <article className="product-card">
        <img src={product.image} alt={product.name} />
        <div className="product-info">
          <h3>{product.name}</h3>
          <p>{product.description}</p>
          <span className="price">{product.price}</span>
          <button>장바구니 추가</button>
        </div>
      </article>
    </div>
  );
}
```

```css
.product-card-wrapper {
  container-type: inline-size;
}

/* 기본: 세로 레이아웃 */
.product-card {
  display: flex;
  flex-direction: column;
}

.product-card img {
  width: 100%;
  aspect-ratio: 1;
  object-fit: cover;
}

/* 컨테이너가 넓을 때: 가로 레이아웃 */
@container (min-width: 480px) {
  .product-card {
    flex-direction: row;
    gap: 1.5rem;
  }

  .product-card img {
    width: 200px;
    aspect-ratio: auto;
    flex-shrink: 0;
  }
}

/* 더 넓을 때: 더 큰 타이포그래피 */
@container (min-width: 720px) {
  .product-card h3 {
    font-size: 1.5rem;
  }

  .product-card .price {
    font-size: 1.25rem;
  }
}
```

## Container Query Units

컨테이너 크기를 단위로 사용할 수 있습니다.

```css
.card {
  container-type: inline-size;
}

.card-title {
  /* 컨테이너 너비의 5% */
  font-size: clamp(1rem, 5cqi, 2rem);
  /* cqi = container query inline size */
  /* cqb = container query block size */
  /* cqw = container query width */
  /* cqh = container query height */
}
```

## Tailwind CSS와 Container Queries

Tailwind CSS v3.2+에서 `@tailwindcss/container-queries` 플러그인으로 지원합니다.

```bash
npm install @tailwindcss/container-queries
```

Tailwind v4에서는 기본 내장:

```html
<!-- 컨테이너 정의 -->
<div class="@container">
  <!-- 컨테이너 너비에 따른 스타일 -->
  <div class="@sm:grid @sm:grid-cols-2 @lg:grid-cols-3">
    <!-- 콘텐츠 -->
  </div>
</div>
```

```html
<!-- 실전 예제: 반응형 카드 그리드 -->
<div class="@container">
  <div class="flex flex-col @md:flex-row gap-4">
    <img class="w-full @md:w-48 rounded-lg" src="..." />
    <div>
      <h3 class="text-lg @lg:text-xl font-bold">제품명</h3>
      <p class="text-sm @md:text-base text-gray-600">설명...</p>
    </div>
  </div>
</div>
```

## 중첩 컨테이너

```css
/* 외부 컨테이너 */
.layout {
  container-type: inline-size;
  container-name: layout;
}

/* 내부 컨테이너 */
.sidebar {
  container-type: inline-size;
  container-name: sidebar;
}

/* 각각 독립적으로 반응 */
@container sidebar (max-width: 250px) {
  .nav-item span {
    display: none; /* 아이콘만 표시 */
  }
}

@container layout (min-width: 1200px) {
  .main-grid {
    grid-template-columns: 1fr 3fr; /* 2열 레이아웃 */
  }
}
```

## 미디어 쿼리 vs Container Queries 선택 기준

| 상황 | 사용할 것 |
|------|-----------|
| 뷰포트 크기에 따른 전체 레이아웃 변경 | 미디어 쿼리 |
| 컴포넌트 위치에 따른 내부 레이아웃 | Container Queries |
| 사이드바/메인 영역 구분 | Container Queries |
| 인쇄 스타일 | 미디어 쿼리 |
| 다크/라이트 모드 | 미디어 쿼리 또는 CSS 변수 |

Container Queries는 컴포넌트 기반 개발의 패러다임과 완벽하게 맞습니다. 컴포넌트가 어디에 배치되든 올바르게 표시되는 진정한 재사용 가능한 UI를 만들 수 있습니다.
