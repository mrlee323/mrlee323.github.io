## GitHub Copilot vs Cursor

AI 코딩 도구 시장에서 가장 많이 사용되는 두 도구, GitHub Copilot과 Cursor를 실제 프론트엔드 개발 워크플로우에서 비교해 보겠습니다.

## GitHub Copilot

### 강점

**1. VS Code 네이티브 통합**

VS Code에 완벽하게 통합되어 기존 워크플로우를 바꿀 필요가 없습니다.

**2. 인라인 제안의 정확도**

타이핑 중 나타나는 인라인 제안이 매우 정확합니다. 특히 반복 패턴 코드에서 탁월합니다.

```typescript
// 이런 패턴을 한 번 작성하면
const { data: users } = useQuery({
  queryKey: ['users'],
  queryFn: fetchUsers,
});

// 다음에 비슷한 걸 작성할 때 Copilot이 자동으로 채워줌
const { data: products } = useQuery({
  // Copilot: queryKey: ['products'], queryFn: fetchProducts,
});
```

**3. Copilot Chat**

복잡한 문제를 채팅으로 해결할 수 있습니다.

```
/fix 이 TypeScript 에러를 어떻게 해결하나요?
/test 이 함수에 대한 테스트를 작성해줘
/explain 이 코드가 무엇을 하는지 설명해줘
/doc JSDoc 주석을 추가해줘
```

### 약점

- 프로젝트 전체 컨텍스트 이해가 제한적
- 대화 내역이 짧음

## Cursor

Cursor는 VS Code 포크로 AI 기능을 핵심에 두고 설계된 IDE입니다.

### 강점

**1. 긴 컨텍스트 대화**

파일 여러 개를 동시에 참조하며 대화할 수 있습니다.

```
@file src/components/UserCard.tsx
@file src/hooks/useUser.ts
@file src/types/user.ts

UserCard 컴포넌트를 리팩토링해서 
useUser 훅과 User 타입을 사용하도록 업데이트해줘
```

**2. Codebase 인덱싱**

프로젝트 전체를 인덱싱해서 "이 프로젝트에서 인증은 어떻게 처리하나요?"같은 질문에 정확히 답변합니다.

**3. Composer 기능**

여러 파일에 걸친 대규모 변경을 한 번에 처리합니다.

```
"사용자 프로필 편집 기능을 추가해줘.
- ProfileEditModal 컴포넌트 생성
- useUpdateProfile 훅 추가
- API 클라이언트에 updateProfile 메서드 추가
- UserProfile 페이지에 편집 버튼 추가"
```

Cursor가 자동으로 4개의 파일을 변경하고 미리보기를 보여줍니다.

### 약점

- 유료 플랜이 비쌈 (월 $20+)
- VS Code 기반이지만 일부 확장 프로그램 호환성 문제
- 처음 적응 시간이 필요

## 실전 활용 시나리오

### 시나리오 1: 새 피처 개발

**Cursor가 유리한 경우:**

새로운 기능 전체를 처음부터 구현할 때, 여러 파일에 걸쳐 일관된 변경이 필요할 때 Cursor의 Composer가 강력합니다.

```
Cursor Composer에서:

"Next.js 15 App Router에서 포스트 즐겨찾기 기능을 구현해줘.
- app/api/favorites/route.ts: GET, POST, DELETE API
- src/hooks/useFavorites.ts: React Query 훅
- src/components/FavoriteButton.tsx: 토글 버튼 컴포넌트
- 낙관적 업데이트 포함"
```

### 시나리오 2: 버그 수정

**GitHub Copilot이 유리한 경우:**

오류 메시지를 Copilot Chat에 붙여넣으면 원인과 해결책을 빠르게 제시합니다.

```
오류: TypeError: Cannot read properties of undefined (reading 'map')
  at UserList (UserList.tsx:15)

이 오류의 원인과 해결책을 알려줘
```

### 시나리오 3: 코드 리뷰

두 도구 모두 `@workspace` 또는 파일 참조로 코드 리뷰를 요청할 수 있습니다.

```typescript
// 이 코드의 잠재적 문제점을 찾아줘

export function useDebounce<T>(value: T, delay: number): T {
  const [debouncedValue, setDebouncedValue] = useState<T>(value);

  useEffect(() => {
    const timer = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(timer);
    };
  }, [value, delay]);

  return debouncedValue;
}
```

## 생산성 극대화 팁

### 1. 좋은 프롬프트 패턴

```
// ❌ 모호한 프롬프트
"컴포넌트 만들어줘"

// ✅ 구체적인 프롬프트
"TypeScript를 사용해서 SearchInput 컴포넌트를 만들어줘.
- debounce 300ms 적용
- 로딩 상태 표시
- 클리어 버튼
- placeholder prop
- onChange callback (debounced 값 전달)"
```

### 2. 주석으로 의도 명확히 하기

```typescript
// Copilot은 주석을 읽고 구현을 제안함

// 사용자 목록을 페이지네이션으로 가져오는 커스텀 훅
// - pageSize: 페이지당 아이템 수 (기본값: 20)
// - filters: 검색 필터 (이름, 역할)
// - 무한 스크롤 지원
function useUserList(options: UserListOptions) {
  // Copilot이 여기서 구현을 제안
}
```

### 3. 반복 패턴 학습시키기

Copilot은 현재 파일의 패턴을 학습합니다. 파일 상단에 비슷한 예시를 두면 일관된 코드를 생성합니다.

## 두 도구를 함께 사용하기

실무에서는 두 도구를 목적에 맞게 함께 사용하는 것이 가장 효과적입니다:

| 작업 | 추천 도구 |
|------|-----------|
| 새 기능 전체 구현 | Cursor Composer |
| 빠른 인라인 자동완성 | GitHub Copilot |
| 코드 설명/리뷰 | 둘 다 |
| 버그 디버깅 | GitHub Copilot Chat |
| 대규모 리팩토링 | Cursor |

AI 도구는 활용할수록 더 잘 다루게 됩니다. 처음엔 어색하더라도 꾸준히 사용하다 보면 개발 속도가 눈에 띄게 빨라지는 것을 경험하게 됩니다.
